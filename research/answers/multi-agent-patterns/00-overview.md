# Multi-Agent Patterns: Overview

Bead: `cr-urcqp0` (Researcher: multi-agent system patterns survey)

Date: 2026-05-04

## Thesis

The surveyed systems collapse to a small set of reusable primitives: role-specialized prompts, a shared state or message substrate, a task or graph scheduler, a tool/action loop, a manager/router, and a final synthesis step. AutoGPT's agent loop gathers directives and commands, builds a prompt, calls an LLM, parses an action, and executes one or more tools; this is the basic autonomous-agent substrate rather than a council substrate (`/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:266-371`, `/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:373-460`). BabyAGI's early form is a task queue with execution, task creation, prioritization, and vector-memory context, all cycling in one loop (`/git/github.com/yoheinakajima/babyagi`, commit `a37ca09`, `classic/babyagi.py:46-94`, `/git/github.com/yoheinakajima/babyagi`, commit `a37ca09`, `classic/babyagi.py:96-136`). MetaGPT turns roles into a team inside an environment with publish/observe message routing and concurrent role runs (`/git/github.com/geekan/MetaGPT/metagpt/team.py:32-36`, `/git/github.com/geekan/MetaGPT/metagpt/environment/base_env.py:175-212`). CrewAI productizes roles plus tasks, with sequential and hierarchical processes and a manager agent that delegates via tools (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/process.py:4-11`, `/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/crew.py:1397-1431`). AutoGen implements group chat as a runtime manager that broadcasts starts, updates a message thread, applies termination, selects next speakers, and publishes requests to participant topics (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/teams/_group_chat/_base_group_chat_manager.py:86-193`). LangGraph exposes the state-machine version of the same idea: nodes read and write shared state, reducers merge parallel writes, edges and conditional edges schedule flow, and graphs must be compiled before execution (`/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:119-133`, `/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:837-930`).

Criopolis is best classified as a durable deliberative council plus a cascade ledger. The mayor prompt defines five council seats, tool agents, an always-on majordomo, and an editorial loop of framing, slinging focused beads, reading closes, pushing back, and synthesizing a defended position (`agents/mayor/prompt.template.md:73-130`). The majordomo prompt defines the operational split: cascade receiver, state tracker, routine dispatcher, first-pass synthesizer, with mayor retaining final editorial voice (`agents/majordomo/prompt.template.md:3-12`, `agents/majordomo/prompt.template.md:34-50`, `agents/majordomo/prompt.template.md:69-108`). That durable bead/mail architecture is closer to a civic workflow engine than to a transient group chat.

## Map of the Territory

| Pattern | Core idea | Strong examples | Criopolis relation |
|---|---|---|---|
| Autonomous tool loop | One agent repeatedly reasons, calls tools, observes results, and continues | AutoGPT proposes actions and executes tools (`/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:266-371`, `/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:373-460`); Swarm loops completion, tool calls, context updates, and possible active-agent switch (`/git/github.com/openai/swarm/swarm/core.py:231-292`) | Tool agents such as researcher are autonomous workers, but the city itself is not one loop (`agents/researcher/prompt.template.md:74-117`) |
| Queue cascade | Pop work, execute, create/prioritize follow-up work | BabyAGI's original loop pops a task, executes it, stores result, creates new tasks, and reprioritizes (`/git/github.com/yoheinakajima/babyagi`, commit `a37ca09`, `classic/babyagi.py:96-136`) | Beads are durable work units; close plus mail is the city's explicit cascade primitive (`agents/researcher/prompt.template.md:91-117`) |
| Role company | Roles cooperate through a shared environment and SOP-like flow | MetaGPT's `Team` owns roles, SOP, and environment (`/git/github.com/geekan/MetaGPT/metagpt/team.py:32-36`); its environment routes messages to roles and runs non-idle roles concurrently (`/git/github.com/geekan/MetaGPT/metagpt/environment/base_env.py:175-212`) | Criopolis also has named roles, but philosophical council seats are evaluative disciplines rather than software-company jobs (`agents/mayor/prompt.template.md:83-104`) |
| Manager-worker hierarchy | A central manager delegates to specialized workers | CrewAI requires a manager LLM or manager agent for hierarchical process and creates a manager with delegation tools (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/crew.py:627-649`, `/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/crew.py:1401-1431`); Anthropic Research uses a lead agent and parallel subagents (Anthropic, "How we built our multi-agent research system", 2025-06-13, https://www.anthropic.com/engineering/multi-agent-research-system) | Mayor/majordomo split is manager-like, but mayor is editor and majordomo is ledger/dispatcher, not a single omnipotent manager (`agents/mayor/prompt.template.md:121-130`, `agents/majordomo/prompt.template.md:69-108`) |
| Group chat | A manager selects speakers and routes messages through runtime topics | AutoGen group chat manager updates message thread and requests next speakers (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/teams/_group_chat/_base_group_chat_manager.py:86-193`) | Criopolis avoids an always-open chat room; it uses closed beads and notes for durable replies (`agents/researcher/prompt.template.md:91-117`) |
| Handoff | A tool result or message changes the active agent | Swarm converts a tool returning an `Agent` into a `Result(agent=...)` and switches active agent in `run` (`/git/github.com/openai/swarm/swarm/core.py:71-87`, `/git/github.com/openai/swarm/swarm/core.py:279-287`) | Criopolis routes by bead assignment and mail, not direct in-conversation transfer (`agents/mayor/prompt.template.md:109-130`) |
| Explicit graph | Nodes and edges encode state, sequencing, fanout, join, and conditional control | LangGraph `StateGraph` defines shared state, reducers, edges, and conditional edges (`/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:119-133`, `/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:837-930`) | Criopolis has implicit state in beads and mail; it could benefit from explicit round-state graphs |
| Debate/dialectic | Multiple agents argue or inspect from different frames; a judge/synthesizer decides | CAMEL `RolePlaying` alternates user and assistant agents and can include a critic (`/git/github.com/camel-ai/camel/camel/societies/role_playing.py:36-90`, `/git/github.com/camel-ai/camel/camel/societies/role_playing.py:631-710`); multiagent debate has multiple model instances debate over rounds (Du et al., "Improving Factuality and Reasoning in Language Models through Multiagent Debate", 2023-05-23, https://arxiv.org/abs/2305.14325) | The high council is the strongest Criopolis identity: five disciplined reviewers whose disagreement mayor must edit into a stance (`agents/mayor/prompt.template.md:73-130`) |

## Opinionated Conclusions

1. Criopolis should preserve the mayor/majordomo split. Existing systems usually merge orchestration, state tracking, and final synthesis in one manager or runtime; Criopolis separates editorial judgment from ledger and cascade handling (`agents/mayor/prompt.template.md:121-130`, `agents/majordomo/prompt.template.md:96-108`).

2. Criopolis should make council-round state more explicit. LangGraph's `StateGraph` shows a clean primitive for nodes reading/writing state and reducers merging parallel node outputs (`/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:119-133`), while Criopolis currently relies on bead status, mail, and prompt discipline for equivalent state (`agents/researcher/prompt.template.md:91-117`).

3. Criopolis should use parallelism only where independence is real. Anthropic's Research system reports strong value for breadth-first independent research and warns that domains with many dependencies fit poorly today (Anthropic, "How we built our multi-agent research system", 2025-06-13, https://www.anthropic.com/engineering/multi-agent-research-system). Claude Code's subagent docs make the same split: parallel research for independent investigations, chaining for dependent phases, and main conversation for phases that share significant context (Anthropic, "Create custom subagents", accessed 2026-05-04, https://code.claude.com/docs/en/subagents).

4. Criopolis should add validation filters for review outputs. Claude Code's code-review plugin launches parallel issue-finding agents, then launches validation agents and filters out unvalidated issues before output (`/git/github.com/anthropics/claude-code/plugins/code-review/commands/code-review.md:30-57`). That is directly useful for council and code-review beads where false positives are expensive.

5. Criopolis should not over-index on consensus. Self-consistency samples multiple reasoning paths and selects the most consistent answer (Wang et al., "Self-Consistency Improves Chain of Thought Reasoning in Language Models", 2022-03-21, https://arxiv.org/abs/2203.11171), while multiagent debate asks model instances to debate toward a common answer (Du et al., "Improving Factuality and Reasoning in Language Models through Multiagent Debate", 2023-05-23, https://arxiv.org/abs/2305.14325). Those are useful but not identical to Criopolis's council; Satya, Viveka, Dharma, Prayoga, and Rasa are not equal voters over the same criterion, because each prompt demands a different evidence surface and failure mode (`agents/satya/prompt.template.md:71-101`, `agents/viveka/prompt.template.md:71-92`, `agents/dharma/prompt.template.md:71-92`, `agents/prayoga/prompt.template.md:71-85`, `agents/rasa/prompt.template.md:71-92`).
