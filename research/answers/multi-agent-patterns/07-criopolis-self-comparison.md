# Criopolis Self-Comparison

## Classification

Criopolis is a durable deliberative council plus tool-agent workflow. It is not primarily an autonomous loop like AutoGPT (`/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:266-460`), not primarily a task queue like BabyAGI (`/git/github.com/yoheinakajima/babyagi`, commit `a37ca09`, `classic/babyagi.py:96-136`), not primarily a group chat like AutoGen (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/teams/_group_chat/_base_group_chat_manager.py:86-193`), and not primarily a handoff loop like Swarm (`/git/github.com/openai/swarm/swarm/core.py:231-292`). The mayor prompt defines a high council, tool agents, majordomo, and an editorial loop (`agents/mayor/prompt.template.md:73-130`). The researcher prompt defines durable bead update/close/mail mechanics (`agents/researcher/prompt.template.md:91-117`). The majordomo prompt defines a persistent ledger and dispatcher role (`agents/majordomo/prompt.template.md:3-108`).

## Borrowed or Rediscovered

Criopolis borrows or independently rediscovers role specialization. MetaGPT uses roles and SOPs in a team (`/git/github.com/geekan/MetaGPT/metagpt/team.py:32-36`), CrewAI selects coworkers by role (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/tools/agent_tools/base_agent_tools.py:70-125`), Claude Code subagents are specialized assistants with custom prompts and tools (Anthropic, "Create custom subagents", accessed 2026-05-04, https://code.claude.com/docs/en/subagents), and Criopolis defines council seats plus tool agents (`agents/mayor/prompt.template.md:83-104`).

Criopolis borrows manager-worker coordination but splits it. CrewAI creates or uses a manager agent for hierarchical process (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/crew.py:627-649`, `/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/crew.py:1401-1431`). Anthropic Research uses a lead agent with parallel specialized subagents (Anthropic, "How we built our multi-agent research system", 2025-06-13, https://www.anthropic.com/engineering/multi-agent-research-system). Criopolis splits lead/editor and ledger/dispatcher between mayor and majordomo (`agents/mayor/prompt.template.md:121-130`, `agents/majordomo/prompt.template.md:69-108`).

Criopolis borrows pub/sub/event thinking but makes it durable. MetaGPT publishes messages to roles and records history (`/git/github.com/geekan/MetaGPT/metagpt/environment/base_env.py:175-195`). AutoGen publishes group chat events to output/group/speaker topics (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/teams/_group_chat/_base_group_chat_manager.py:104-193`). Criopolis uses `bd update`, `bd close`, and `gc mail send --notify majordomo` as the cascade primitive (`agents/researcher/prompt.template.md:91-117`).

Criopolis borrows final-arbiter synthesis. AutoGen's Society-of-Mind synthesizes an inner team result (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/agents/_society_of_mind_agent.py:168-250`). Anthropic Research has the lead agent synthesize subagent findings (Anthropic, "How we built our multi-agent research system", 2025-06-13, https://www.anthropic.com/engineering/multi-agent-research-system). Criopolis makes mayor final editor, with majordomo producing first-pass digests only (`agents/mayor/prompt.template.md:121-130`, `agents/majordomo/prompt.template.md:96-108`).

## What Appears Novel in the Surveyed Set

The exact combination of durable bead/mail workflow, mayor/majordomo separation, and five philosophical council seats was not found in the surveyed source. AutoGen has group-chat managers and nested teams (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/teams/_group_chat/_base_group_chat_manager.py:86-193`, `/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/agents/_society_of_mind_agent.py:168-250`). LangGraph has explicit state graphs (`/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:119-133`). Claude Code has subagents and plugin workflows (`/git/github.com/anthropics/claude-code/plugins/README.md:3-25`). CAMEL has role-playing and workforce societies (`/git/github.com/camel-ai/camel/camel/societies/role_playing.py:36-90`, `/git/github.com/camel-ai/camel/camel/societies/workforce/workforce.py:175-186`). None of those surveyed implementations define Criopolis's five-seat council with Satya/Viveka/Dharma/Prayoga/Rasa output contracts plus a persistent majordomo ledger (`agents/mayor/prompt.template.md:83-104`, `agents/majordomo/prompt.template.md:3-108`, `agents/satya/prompt.template.md:71-101`, `agents/viveka/prompt.template.md:71-92`, `agents/dharma/prompt.template.md:71-92`, `agents/prayoga/prompt.template.md:71-85`, `agents/rasa/prompt.template.md:71-92`).

This is a bounded novelty claim. It means "not present in the surveyed code and web sources," not "globally unprecedented."

## Strong Design Choices to Keep

Keep bead durability. The researcher prompt's cascade pattern makes work state explicit and recoverable: update notes, close bead, notify majordomo (`agents/researcher/prompt.template.md:91-117`). This is more reliable than transient chat-only coordination when work crosses agents and sessions.

Keep majordomo as noise absorber. The majordomo prompt explicitly says the role exists to absorb city noise and give mayor decision-ready digests only when major events happen (`agents/majordomo/prompt.template.md:29-50`). This matches Anthropic's warning that multi-agent systems can produce excessive updates and coordination complexity (Anthropic, "How we built our multi-agent research system", 2025-06-13, https://www.anthropic.com/engineering/multi-agent-research-system).

Keep source-citation discipline. Researcher requires source cloning, call-graph tracing, and file:line citations (`agents/researcher/prompt.template.md:27-44`). Anthropic Research's CitationAgent shows the same high-level need for citation processing after synthesis (Anthropic, "How we built our multi-agent research system", 2025-06-13, https://www.anthropic.com/engineering/multi-agent-research-system).

Keep differentiated seats. The council is valuable because Satya, Viveka, Dharma, Prayoga, and Rasa inspect different failure modes (`agents/satya/prompt.template.md:71-101`, `agents/viveka/prompt.template.md:71-92`, `agents/dharma/prompt.template.md:71-92`, `agents/prayoga/prompt.template.md:71-85`, `agents/rasa/prompt.template.md:71-92`). Turning them into generic voters would discard the architecture's strongest feature.

## Changes to Consider

Add explicit round-state metadata. LangGraph shows the value of explicit nodes, edges, joins, and conditional routing (`/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:119-133`, `/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:837-930`). Criopolis can keep beads but encode round states such as `framed`, `researching`, `seat-review`, `joined`, `majordomo-digest`, `mayor-synthesis`, and `archived`.

Add synthesis mode metadata. A bead could declare whether the expected synthesis is final-arbiter, role-owned sections, validation filter, critic-refiner, or sequential cascade. These modes are all evidenced in surveyed systems: mayor final-arbiter (`agents/mayor/prompt.template.md:121-130`), Claude Code validation filter (`/git/github.com/anthropics/claude-code/plugins/code-review/commands/code-review.md:30-57`), CrewAI sequential/hierarchical process (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/process.py:4-11`), and CAMEL critic-capable role play (`/git/github.com/camel-ai/camel/camel/societies/role_playing.py:36-90`).

Add validation passes for high-risk findings. Claude Code's code-review command filters unvalidated issues after validation subagents inspect them (`/git/github.com/anthropics/claude-code/plugins/code-review/commands/code-review.md:55-57`). Criopolis should use the same pattern when council objections imply code changes, policy changes, or claims about external source.

Use parallelism as a declared dependency choice. Anthropic Research says multi-agent systems excel at breadth-first queries with independent directions and large context, but not when all agents need the same context or many dependencies (Anthropic, "How we built our multi-agent research system", 2025-06-13, https://www.anthropic.com/engineering/multi-agent-research-system). Claude Code docs say parallel research works best when paths do not depend on each other and warns of context cost (Anthropic, "Create custom subagents", accessed 2026-05-04, https://code.claude.com/docs/en/subagents). Criopolis should mark fanout beads with the dependency reason.

Do not add market/auction allocation yet. Contract Net is real prior art for negotiated task allocation (Smith, "The Contract Net Protocol: High-Level Communication and Control in a Distributed Problem Solver", 1980-12, https://doi.org/10.1109/TC.1980.1675516), but current Criopolis scarce resources are attention and correctness, not a pool of interchangeable workers bidding on tasks. Majordomo already manages attention by absorbing routine events (`agents/majordomo/prompt.template.md:31-50`).

## Bottom Line

Criopolis should evolve toward "durable council workflow with explicit state," not toward "chat room with many agents." AutoGen is valuable for speaker policy, LangGraph for explicit state, Claude Code for bounded subagent delegation, Anthropic Research for parallel research economics, and CAMEL for role-play/dialectic. Criopolis's own durable bead cascade and mayor/majordomo/council split are the architecture to protect (`agents/mayor/prompt.template.md:73-130`, `agents/majordomo/prompt.template.md:3-108`, `agents/researcher/prompt.template.md:91-117`).
