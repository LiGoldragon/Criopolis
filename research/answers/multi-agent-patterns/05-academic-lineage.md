# Academic and Theoretical Lineage

## Classical Intelligent Agents

Wooldridge and Jennings's "Intelligent Agents: Theory and Practice" is a foundational 1995 survey that frames agents through autonomy, reactivity, pro-activeness, and social ability (Wooldridge and Jennings, "Intelligent Agents: Theory and Practice", 1995, https://www.cs.ox.ac.uk/people/michael.wooldridge/pubs/ker95/ker95-html.html). Criopolis maps cleanly to the social-ability part of that lineage because it organizes multiple named agents and explicit communication channels, not just one autonomous loop (`agents/mayor/prompt.template.md:73-130`, `agents/researcher/prompt.template.md:91-117`).

BDI architectures provide the belief-desire-intention lineage for agents that maintain informational state, goals, and committed plans. Rao and Georgeff's "BDI Agents: From Theory to Practice" appeared at ICMAS 1995 and presents BDI as a practical rational-agent architecture (Rao and Georgeff, "BDI Agents: From Theory to Practice", 1995-04, https://www.math.pku.edu.cn/teachers/linzq/teaching/stm/references/BDI%20Agents%20From%20Theory%20to%20Practice.pdf). In modern LLM systems, BDI shows up less as formal modal logic and more as task state, plans, tool loops, and memory. AutoGPT's state/task/directives/actions loop is a practical descendant of this architecture style (`/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:266-371`, `/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:373-460`).

## Coordination Theory

Malone and Crowston define coordination theory around managing dependencies among activities (Malone and Crowston, "The Interdisciplinary Study of Coordination", ACM Computing Surveys, 1994-03, https://crowston.syr.edu/bibcite/reference/6660; open PDF https://crowston.syr.edu/sites/default/files/acmcs94.pdf). This is the cleanest theoretical frame for comparing agent systems because the hard problem is rarely "how many agents?" and usually "which dependencies are real?" LangGraph's graph edges and reducer-backed shared state make dependencies explicit (`/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:119-133`, `/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:837-930`). Criopolis's bead/mail cascade makes dependencies durable but mostly implicit in bead metadata and prompt discipline (`agents/researcher/prompt.template.md:91-117`, `agents/majordomo/prompt.template.md:34-94`).

## Distributed Problem Solving and Contract Net

Smith's Contract Net Protocol is the classical market/auction reference: task distribution is handled through negotiation between nodes with tasks and nodes able to execute them (Smith, "The Contract Net Protocol: High-Level Communication and Control in a Distributed Problem Solver", IEEE Transactions on Computers, 1980-12, https://doi.org/10.1109/TC.1980.1675516; open copy https://cse-robotics.engr.tamu.edu/dshell/cs631/papers/smith80contract.pdf). Current LLM frameworks surveyed here mostly use manager assignment, speaker selection, or graph routing instead of bidding. CrewAI's delegation chooses a coworker by role match (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/tools/agent_tools/base_agent_tools.py:70-125`), and Swarm switches active agents through tool return values rather than bids (`/git/github.com/openai/swarm/swarm/core.py:71-87`, `/git/github.com/openai/swarm/swarm/core.py:279-287`).

## Society of Mind and Agent Societies

Minsky's "Society of Mind" is the older conceptual ancestor for mind-as-many-specialists, and modern LLM systems reuse the phrase and pattern even when they do not implement Minsky's theory (Minsky, "The Society of Mind", 1986, book reference: https://web.media.mit.edu/~minsky/som.html). AutoGen includes a `SocietyOfMindAgent` that wraps an inner team and then synthesizes the team's output into one final response (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/agents/_society_of_mind_agent.py:168-250`). The multiagent debate paper explicitly invokes a "society of minds" approach for multiple LLM instances debating toward better answers (Du et al., "Improving Factuality and Reasoning in Language Models through Multiagent Debate", 2023-05-23, https://arxiv.org/abs/2305.14325). CAMEL presents communicative agents as a framework for studying LLM societies (Li et al., "CAMEL: Communicative Agents for 'Mind' Exploration of Large Language Model Society", 2023-03-31, https://arxiv.org/abs/2303.17760).

Criopolis is society-of-mind-like in specialization, but it is not a model of one mind. It is a civic apparatus with named offices, durable beads, and an editor (`agents/mayor/prompt.template.md:73-130`, `agents/majordomo/prompt.template.md:3-108`).

## ReAct and Tool-Using Language Agents

ReAct introduced an influential LLM-agent pattern: interleaving reasoning traces and task-specific actions, so reasoning can track plans and actions can gather external information (Yao et al., "ReAct: Synergizing Reasoning and Acting in Language Models", 2022-10-06, https://arxiv.org/abs/2210.03629). AutoGPT's propose/complete/execute cycle is a productized version of an interleaved thought/action loop, with function specs and tool execution surrounding LLM calls (`/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:266-460`). BabyAGI's current draft `react_agent` explicitly builds tools and loops over tool calls until final answer or iteration limit (`/git/github.com/yoheinakajima/babyagi/babyagi/functionz/packs/drafts/react_agent.py:28-190`).

Criopolis uses ReAct-like loops inside tool agents, but the city-level architecture is not just ReAct. The city-level primitive is bead-based delegation and synthesis (`agents/researcher/prompt.template.md:91-117`, `agents/mayor/prompt.template.md:109-130`).

## Reflection, Self-Consistency, and Search

Reflexion moves learning into verbal memory: agents reflect on feedback and keep those reflections for future trials (Shinn et al., "Reflexion: Language Agents with Verbal Reinforcement Learning", 2023-03-20, https://arxiv.org/abs/2303.11366). Self-consistency samples multiple reasoning paths and selects the most consistent answer (Wang et al., "Self-Consistency Improves Chain of Thought Reasoning in Language Models", 2022-03-21, https://arxiv.org/abs/2203.11171). AutoGPT exposes `reflexion` and `tree_of_thoughts` prompt strategies in source, showing those academic ideas moving into agent frameworks (`/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:613-625`).

For Criopolis, these patterns are useful for local reasoning robustness but should not replace council synthesis. Council seats are role-specialized checks; self-consistency is many samples of a similar reasoning process (`agents/satya/prompt.template.md:71-101`, `agents/viveka/prompt.template.md:71-92`, `agents/dharma/prompt.template.md:71-92`).

## Generative Agents, Memory, and Planning

Stanford's Generative Agents paper describes agents that store experiences in natural language, synthesize reflections, retrieve memories dynamically, and plan behavior (Park et al., "Generative Agents: Interactive Simulacra of Human Behavior", 2023-04-07, https://arxiv.org/abs/2304.03442). BabyAGI's original Pinecone context agent is a simpler memory-retrieval ancestor in LLM-agent practice (`/git/github.com/yoheinakajima/babyagi`, commit `a37ca09`, `classic/babyagi.py:86-94`). CAMEL Workforce's shared memory option gives another practical memory substrate for coordinated work (`/git/github.com/camel-ai/camel/camel/societies/workforce/workforce.py:224-231`). Criopolis's durable memory is operational rather than psychological: bead notes and mail preserve who did what and what should happen next (`agents/researcher/prompt.template.md:91-117`, `agents/majordomo/prompt.template.md:34-50`).

## Voyager and Embodied Skill Libraries

Voyager uses an automatic curriculum, an ever-growing skill library of executable code, and iterative prompting with environment feedback and self-verification for Minecraft (Wang et al., "Voyager: An Open-Ended Embodied Agent with Large Language Models", 2023-05-25, https://arxiv.org/abs/2305.16291). This lineage matters for Criopolis only when code-writer agents start accumulating reusable skills or repair patterns. The current Criopolis prompts define tool agents and output rigs but do not define a skill library equivalent to Voyager's (`agents/mayor/prompt.template.md:91-99`, `agents/researcher/prompt.template.md:27-44`).

## Modern Framework Papers

MetaGPT's paper says the framework encodes standardized operating procedures into prompt sequences and uses an assembly-line paradigm with diverse roles (Hong et al., "MetaGPT: Meta Programming for A Multi-Agent Collaborative Framework", 2023-08-01, https://arxiv.org/abs/2308.00352). The cloned source reflects this in `Team` roles, SOP language, environment, role hiring, and round-based environment execution (`/git/github.com/geekan/MetaGPT/metagpt/team.py:32-138`, `/git/github.com/geekan/MetaGPT/metagpt/environment/base_env.py:175-212`). CAMEL's paper frames communicative role-playing agents as a way to study cooperative behavior in LLM societies (Li et al., "CAMEL: Communicative Agents for 'Mind' Exploration of Large Language Model Society", 2023-03-31, https://arxiv.org/abs/2303.17760), and the source implements two-agent role playing plus workforce orchestration (`/git/github.com/camel-ai/camel/camel/societies/role_playing.py:36-90`, `/git/github.com/camel-ai/camel/camel/societies/workforce/workforce.py:175-186`).

## Agentic Workflow Guidance

Anthropic's "Building effective agents" distinguishes workflows from agents: workflows orchestrate LLMs/tools through predefined code paths, while agents dynamically direct process and tool use (Anthropic, "Building effective agents", 2024-12-19, https://www.anthropic.com/engineering/building-effective-agents). The post recommends simple composable patterns over complex frameworks for many production systems (Anthropic, "Building effective agents", 2024-12-19, https://www.anthropic.com/engineering/building-effective-agents). LangChain's multi-agent docs similarly say not every complex task needs multi-agent architecture and identify context management, distributed development, and parallelization as common reasons to use it (LangChain, "Multi-agent", accessed 2026-05-04, https://docs.langchain.com/oss/python/langchain/multi-agent/index).

Criopolis has a defensible reason for multi-agent architecture: its council seats carry genuinely different evaluation contracts, and majordomo protects mayor attention by absorbing routine cascade noise (`agents/mayor/prompt.template.md:83-130`, `agents/majordomo/prompt.template.md:31-50`).
