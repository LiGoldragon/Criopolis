# Role Taxonomies

## Stable Role Families

The surveyed systems instantiate roles that fall into recurring families: orchestrator, dispatcher, planner, executor, researcher, critic/verifier, synthesizer, memory keeper, router, and domain specialist. The exact names differ, but the responsibilities recur in source. Criopolis names mayor as orchestrator/editor, majordomo as persistent assistant and first-pass synthesizer, council seats as evaluative disciplines, and tool agents as researcher/explorer/code-writer (`agents/mayor/prompt.template.md:73-104`, `agents/majordomo/prompt.template.md:3-12`).

## Orchestrator / Editor / Lead Agent

An orchestrator owns the global question and chooses who works next. Criopolis's mayor frames questions, slings focused beads, watches closures, pushes back, and synthesizes a defensible stance (`agents/mayor/prompt.template.md:109-130`). Anthropic's Research system uses a lead agent that analyzes a query, develops a strategy, spawns subagents, decides whether more research is needed, and synthesizes results (Anthropic, "How we built our multi-agent research system", 2025-06-13, https://www.anthropic.com/engineering/multi-agent-research-system). CrewAI's hierarchical process creates or uses a manager agent, sets delegation on it, and gives a generated manager tools for delegating to crew agents (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/crew.py:1401-1431`). AutoGen's group chat manager is an orchestrator at the runtime level: it validates state, publishes starts, updates the thread, checks termination, selects speakers, and requests participant responses (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/teams/_group_chat/_base_group_chat_manager.py:86-193`).

## Dispatcher / State Tracker

A dispatcher turns completed work into next work and watches system state. Criopolis makes this explicit in majordomo: cascade receiver, state tracker, routine dispatcher, first-pass synthesizer, always-on beside mayor (`agents/majordomo/prompt.template.md:3-12`). Majordomo's cascade receiver reads mail, reads closed bead notes, decides absorb/sling/escalate, marks mail read, and updates the state ledger (`agents/majordomo/prompt.template.md:34-50`). Its routine dispatcher handles cascade follow-ups, mayor-delegated dispatches, stuck-work nudges, and dependent dispatches, while refusing council rounds and mayor-level framing (`agents/majordomo/prompt.template.md:69-94`). This explicit separation is not present in CrewAI's manager creation, which combines delegation authority in the manager agent (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/crew.py:1406-1431`), or in AutoGen's group chat manager, which owns runtime speaker transitions (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/teams/_group_chat/_base_group_chat_manager.py:172-193`).

## Planner / Decomposer

Planners decompose work and decide order. BabyAGI's original `task_creation_agent` creates new tasks from the last result and incomplete tasks, and `prioritization_agent` reprioritizes the queue (`/git/github.com/yoheinakajima/babyagi`, commit `a37ca09`, `classic/babyagi.py:46-69`). CAMEL Workforce documents an internal Task Planner Agent that decomposes complex tasks and composes results (`/git/github.com/camel-ai/camel/camel/societies/workforce/workforce.py:181-185`). CAMEL also constructs a default task agent with a "Task Planner" role name (`/git/github.com/camel-ai/camel/camel/societies/workforce/workforce.py:465-480`). AutoGPT supports multiple planning-style prompt strategies, including `rewoo`, `plan_execute`, `reflexion`, `tree_of_thoughts`, and `lats` (`/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:588-633`).

## Executor / Tool User

Executors perform concrete actions. AutoGPT's executor checks permissions, executes one tool directly or multiple tools in parallel, records ReWOO results, and runs an after-execute pipeline (`/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:373-460`). Its parallel executor gathers tool calls concurrently and combines outputs or errors (`/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:499-557`). BabyAGI's original `execution_agent` retrieves context and asks an LLM to perform one task (`/git/github.com/yoheinakajima/babyagi`, commit `a37ca09`, `classic/babyagi.py:70-84`). CrewAI's manager delegation tool resolves a coworker by role, creates a task assigned to that agent, and calls `selected_agent.execute_task` (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/tools/agent_tools/base_agent_tools.py:70-125`).

## Researcher / Explorer

Researcher roles isolate information gathering and citation work. Criopolis defines the researcher as the tool agent that investigates source-code questions, cites file:line, and writes to `research/answers/` (`agents/mayor/prompt.template.md:91-99`). The researcher prompt requires `ghq get`, local source tracing, notes and answers under `research/`, bead updates, closes, and majordomo notification (`agents/researcher/prompt.template.md:27-44`, `agents/researcher/prompt.template.md:74-117`). Claude Code's official docs define Explore as a fast read-only agent for code search and codebase exploration, and Plan as a read-only research agent used in plan mode (Anthropic, "Create custom subagents", accessed 2026-05-04, https://code.claude.com/docs/en/subagents). Claude Code's plugin examples launch 2-3 code-explorer agents in parallel for codebase exploration before the main agent reads identified files (`/git/github.com/anthropics/claude-code/plugins/feature-dev/commands/feature-dev.md:36-53`).

## Critic / Verifier / Reviewer

Critics inspect outputs, reject false positives, or improve proposals. CAMEL `RolePlaying` can include a critic in the loop and names the critic role in the constructor arguments (`/git/github.com/camel-ai/camel/camel/societies/role_playing.py:36-90`). Claude Code's code-review plugin launches independent review agents, then launches validation subagents for each issue and filters out issues not validated (`/git/github.com/anthropics/claude-code/plugins/code-review/commands/code-review.md:30-57`). The same plugin restricts desired findings to high-signal compile, parse, logic, and exact policy violations, which makes the reviewer role adversarial to false positives (`/git/github.com/anthropics/claude-code/plugins/code-review/commands/code-review.md:41-57`). Criopolis's Satya prompt asks for source bones, falsification ledger, claim type, and a vote, making it a truth-verifier rather than a general reviewer (`agents/satya/prompt.template.md:71-101`). Viveka audits terminology and refuses term smuggling (`agents/viveka/prompt.template.md:71-92`), while Prayoga asks what the proposal costs under real use (`agents/prayoga/prompt.template.md:71-85`).

## Synthesizer / Judge / Arbiter

Synthesizers read multiple outputs and produce a final stance or report. Criopolis mayor is explicitly editor rather than stenographer and must synthesize a position that would be defended (`agents/mayor/prompt.template.md:121-130`). Majordomo is the first-pass synthesizer for council rounds and produces a digest for mayor to refine (`agents/majordomo/prompt.template.md:96-108`). AutoGen's Society-of-Mind agent wraps an inner team and then uses a model to synthesize the inner team's result before resetting the inner team (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/agents/_society_of_mind_agent.py:168-250`). CAMEL's role-playing worker summarizes chat history into a task result after the alternating role-play loop (`/git/github.com/camel-ai/camel/camel/societies/workforce/role_playing_worker.py:195-235`). Anthropic Research passes gathered findings through a CitationAgent after lead-agent synthesis to attach citations (Anthropic, "How we built our multi-agent research system", 2025-06-13, https://www.anthropic.com/engineering/multi-agent-research-system).

## Memory Keeper

Memory roles preserve context across long work. BabyAGI's original `context_agent` queries Pinecone and returns sorted prior task metadata as context for execution (`/git/github.com/yoheinakajima/babyagi`, commit `a37ca09`, `classic/babyagi.py:86-94`). CAMEL Workforce can share complete conversation history and function-calling trajectory across SingleAgentWorkers, coordinator, and task planner when `share_memory` is enabled (`/git/github.com/camel-ai/camel/camel/societies/workforce/workforce.py:224-231`). LangGraph's state object is the architectural memory: nodes read state and return partial state updates, with optional reducers per key (`/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:119-133`). Criopolis uses beads, bead notes, and mail as durable memory for work state rather than relying only on chat context (`agents/researcher/prompt.template.md:91-117`, `agents/majordomo/prompt.template.md:34-50`).

## Router / Handoff Agent

Routers decide which specialist receives control. Swarm makes handoff a return value: an `AgentFunction` may return an `Agent`, `Result`, or string/dict, and `handle_function_result` wraps a returned `Agent` as `Result(agent=agent)` (`/git/github.com/openai/swarm/swarm/types.py:11-41`, `/git/github.com/openai/swarm/swarm/core.py:71-87`). Swarm's `run` loop switches `active_agent` when a tool result contains a new agent (`/git/github.com/openai/swarm/swarm/core.py:279-287`). LangChain's multi-agent docs describe subagents, handoffs, skills, router, and custom workflow as first-class patterns (LangChain, "Multi-agent", accessed 2026-05-04, https://docs.langchain.com/oss/python/langchain/multi-agent/index). Criopolis routing is not a tool-return handoff; mayor or majordomo sling beads to named agents, and completion is recorded through bead notes and mail (`agents/mayor/prompt.template.md:109-130`, `agents/majordomo/prompt.template.md:69-94`).

## Domain Specialist / Council Seat

Domain specialists apply narrow criteria. MetaGPT's `Team` is explicitly a set of roles with SOPs and an environment for multi-agent activity (`/git/github.com/geekan/MetaGPT/metagpt/team.py:32-36`). CrewAI agents are selected by role name during manager delegation (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/tools/agent_tools/base_agent_tools.py:80-125`). Claude Code subagents are specialized assistants with their own context window, system prompt, tool access, and permissions, and Claude delegates based on the subagent's description (Anthropic, "Create custom subagents", accessed 2026-05-04, https://code.claude.com/docs/en/subagents). Criopolis council seats are not just domain workers: Satya checks truth and falsification (`agents/satya/prompt.template.md:71-101`), Viveka checks discrimination and conceptual boundaries (`agents/viveka/prompt.template.md:71-92`), Dharma checks right order and absent parties (`agents/dharma/prompt.template.md:71-92`), Prayoga checks application under load (`agents/prayoga/prompt.template.md:71-85`), and Rasa checks felt disclosure and form (`agents/rasa/prompt.template.md:71-92`).

## Practical Taxonomy for Criopolis

For Criopolis architecture decisions, the most useful taxonomy is not "agent count." It is "where authority lives." Evidence supports at least these authority buckets: editorial authority in mayor (`agents/mayor/prompt.template.md:121-130`), ledger/dispatch authority in majordomo (`agents/majordomo/prompt.template.md:34-94`), evidence authority in researcher (`agents/researcher/prompt.template.md:27-44`), criterion authority in council seats (`agents/satya/prompt.template.md:71-101`, `agents/viveka/prompt.template.md:71-92`, `agents/dharma/prompt.template.md:71-92`, `agents/prayoga/prompt.template.md:71-85`, `agents/rasa/prompt.template.md:71-92`), and execution authority in code-writing/tool agents (`agents/mayor/prompt.template.md:91-99`).
