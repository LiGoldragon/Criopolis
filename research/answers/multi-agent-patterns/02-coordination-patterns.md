# Coordination Patterns

## Sequential Cascade

Sequential cascade means one result becomes the next work item. BabyAGI's original loop pops the next task, executes it, stores the result, creates new tasks, appends them, and reprioritizes (`/git/github.com/yoheinakajima/babyagi`, commit `a37ca09`, `classic/babyagi.py:96-136`). CrewAI's sequential process simply calls `_execute_tasks(self.tasks)`, and `_execute_tasks` processes task outputs in order except when async tasks are explicitly launched and joined (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/crew.py:1397-1399`, `/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/crew.py:1441-1508`). Criopolis's researcher prompt defines the durable cascade as bead notes, `bd close`, then `gc mail send --notify majordomo`, making sequencing visible in bead state and mail (`agents/researcher/prompt.template.md:91-117`).

Use sequential cascade when the next task depends on the previous answer. Claude Code's subagent docs explicitly recommend chaining subagents for multi-step workflows where each result is passed to the next subagent (Anthropic, "Create custom subagents", accessed 2026-05-04, https://code.claude.com/docs/en/subagents). Criopolis already encodes this dependency style in the researcher cascade contract: answer, update bead notes, close, then notify majordomo for downstream handling (`agents/researcher/prompt.template.md:91-117`, `agents/majordomo/prompt.template.md:34-50`).

## Parallel Ensemble

Parallel ensemble means independent workers run at the same time and a parent synthesizes results. AutoGPT can execute multiple tools in parallel via `asyncio.gather` and return combined outputs or partial errors (`/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:499-557`). MetaGPT's environment gathers non-idle role futures concurrently (`/git/github.com/geekan/MetaGPT/metagpt/environment/base_env.py:197-212`). Claude Code's feature-dev plugin launches 2-3 code-explorer agents in parallel for different aspects, 2-3 architect agents in parallel for different tradeoff stances, and 3 reviewer agents in parallel for different review lenses (`/git/github.com/anthropics/claude-code/plugins/feature-dev/commands/feature-dev.md:36-53`, `/git/github.com/anthropics/claude-code/plugins/feature-dev/commands/feature-dev.md:73-82`, `/git/github.com/anthropics/claude-code/plugins/feature-dev/commands/feature-dev.md:101-109`).

Anthropic's Research system is the clearest production-style parallel research case: a lead researcher spawns specialized subagents to search different aspects simultaneously and later compiles results (Anthropic, "How we built our multi-agent research system", 2025-06-13, https://www.anthropic.com/engineering/multi-agent-research-system). Claude Code docs give the same rule of thumb: independent investigations can be spawned in parallel, but many detailed returns can consume main context (Anthropic, "Create custom subagents", accessed 2026-05-04, https://code.claude.com/docs/en/subagents). Criopolis should reserve broad parallelism for independent evidence gathering, not tightly coupled implementation phases.

## Hierarchical Manager-Worker

Hierarchy gives one agent authority to create, assign, or request work from subagents. CrewAI's hierarchical process validates that a manager LLM or manager agent exists and forbids putting the manager agent in the regular agents list (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/crew.py:627-649`). If a manager is not supplied, CrewAI creates a manager agent with delegation tools over the crew's agents and `allow_delegation=True` (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/crew.py:1406-1431`). CAMEL Workforce documents a Coordinator Agent for task assignment, a Task Planner Agent for decomposition/composition, and Dynamic Workers created when tasks repeatedly fail (`/git/github.com/camel-ai/camel/camel/societies/workforce/workforce.py:175-186`).

Criopolis has a hierarchy, but it is split. Mayor owns editorial orchestration and final stance (`agents/mayor/prompt.template.md:121-130`). Majordomo owns routine dispatch and state tracking, while being barred from council rounds and mayor-level framing (`agents/majordomo/prompt.template.md:69-94`). This avoids a common manager-worker failure mode: the same manager that assigns work also compresses, judges, and forgets the operational ledger.

## Graph-Based State Machine

Graph-based coordination makes control flow explicit. LangGraph's `StateGraph` nodes communicate by reading and writing shared state, and reducers aggregate writes from multiple nodes (`/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:119-133`). `add_edge` can wait for all start nodes before executing an end node, which gives a native fanout/fan-in join primitive (`/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:837-889`). `add_conditional_edges` lets a node route to one or more destinations or stop at `END` (`/git/github.com/langchain-ai/langgraph/libs/langgraph/langgraph/graph/state.py:891-930`). LangChain docs describe custom workflow as building bespoke execution flows with LangGraph, mixing deterministic logic and agentic behavior (LangChain, "Multi-agent", accessed 2026-05-04, https://docs.langchain.com/oss/python/langchain/multi-agent/index).

Criopolis currently uses a graph implicitly: a research bead closes, majordomo is notified, council review beads become ready, and mayor later synthesizes. The code evidence for that implicit graph is the researcher cascade contract and majordomo's cascade receiver/dispatcher responsibilities (`agents/researcher/prompt.template.md:91-117`, `agents/majordomo/prompt.template.md:34-94`). The recommendation is to keep beads as the durable storage layer but add explicit round metadata for fanout, join, review, synthesis, and archive states.

## Event-Driven Pub/Sub

Event-driven systems route messages or events to subscribers. MetaGPT's environment stores roles, accepts published messages, matches recipients, puts messages into role buffers, and records message history (`/git/github.com/geekan/MetaGPT/metagpt/environment/base_env.py:124-135`, `/git/github.com/geekan/MetaGPT/metagpt/environment/base_env.py:175-195`). AutoGen's group chat manager publishes start messages to output and group topics, then publishes requests to selected speaker topics (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/teams/_group_chat/_base_group_chat_manager.py:104-124`, `/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/teams/_group_chat/_base_group_chat_manager.py:185-193`). Criopolis eventing is bead/mail based: every agent closes a bead and notifies majordomo; majordomo reads mail and bead notes, then absorbs, slings follow-up, or escalates (`agents/researcher/prompt.template.md:91-117`, `agents/majordomo/prompt.template.md:34-50`).

The key tradeoff is durability. AutoGen and MetaGPT route runtime messages (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/teams/_group_chat/_base_group_chat_manager.py:86-193`, `/git/github.com/geekan/MetaGPT/metagpt/environment/base_env.py:175-212`). Criopolis routes durable work objects and mail, so a close survives context loss and can trigger a later review round (`agents/researcher/prompt.template.md:91-117`).

## Handoff

Handoff switches the active agent instead of asking a manager to synthesize all sub-results. Swarm defines an `Agent` with instructions, functions, model, and `parallel_tool_calls`, and a `Result` may carry a next `agent` plus context variables (`/git/github.com/openai/swarm/swarm/types.py:14-41`). In Swarm, returning an `Agent` from a function becomes `Result(agent=agent)`, and the `run` loop switches `active_agent` when a partial response contains an agent (`/git/github.com/openai/swarm/swarm/core.py:71-87`, `/git/github.com/openai/swarm/swarm/core.py:279-287`). LangChain docs list handoffs as a pattern where behavior changes dynamically based on state and tool calls switch agents or adjust prompt/tools (LangChain, "Multi-agent", accessed 2026-05-04, https://docs.langchain.com/oss/python/langchain/multi-agent/index).

Criopolis should be cautious with direct handoff for council work. A handoff transfers control, while council work needs preserved independent criteria and an editorial final stance (`agents/mayor/prompt.template.md:121-130`, `agents/satya/prompt.template.md:71-101`, `agents/rasa/prompt.template.md:71-92`). Handoff is more appropriate for operational triage, for example a support issue moving from researcher to code-writer after evidence is complete.

## Debate / Dialectic

Debate patterns ask multiple agents to expose disagreement before a judge or common answer. CAMEL `RolePlaying` is a two-agent conversation where a user agent responds to an assistant, then the assistant responds to the user; optional task planner and critic roles can be included (`/git/github.com/camel-ai/camel/camel/societies/role_playing.py:36-90`, `/git/github.com/camel-ai/camel/camel/societies/role_playing.py:631-710`). The multiagent debate paper frames a method where multiple language-model instances propose and debate answers over rounds to reach a final answer (Du et al., "Improving Factuality and Reasoning in Language Models through Multiagent Debate", 2023-05-23, https://arxiv.org/abs/2305.14325).

Criopolis is dialectical but not symmetric debate. The five seats inspect a proposition from different disciplines: Satya truth/falsification (`agents/satya/prompt.template.md:71-101`), Viveka discrimination/term boundaries (`agents/viveka/prompt.template.md:71-92`), Dharma right-order/absent parties (`agents/dharma/prompt.template.md:71-92`), Prayoga application under load (`agents/prayoga/prompt.template.md:71-85`), and Rasa felt-disclosure/form (`agents/rasa/prompt.template.md:71-92`). The mayor's job is to push back on real disagreement and synthesize a defensible stance, not to average votes (`agents/mayor/prompt.template.md:121-130`).

## Market / Auction

Market or auction coordination allocates tasks through proposals and awards. The classical Contract Net Protocol describes task distribution through negotiation between nodes with tasks and nodes able to execute them (Smith, "The Contract Net Protocol: High-Level Communication and Control in a Distributed Problem Solver", 1980-12, https://doi.org/10.1109/TC.1980.1675516; open copy: https://cse-robotics.engr.tamu.edu/dshell/cs631/papers/smith80contract.pdf). This pattern was not found as a central primitive in the surveyed LLM-agent source. CrewAI manager delegation selects an agent by matching role name rather than by bidding (`/git/github.com/crewAIInc/crewAI/lib/crewai/src/crewai/tools/agent_tools/base_agent_tools.py:70-125`), and Criopolis slings work to a named agent through mayor/majordomo judgment rather than bids (`agents/mayor/prompt.template.md:109-130`, `agents/majordomo/prompt.template.md:69-94`).

The recommendation is not to import market allocation unless Criopolis develops real contention for scarce resources. The current scarce resource is mayor attention, and majordomo already manages that by absorbing routine cascade noise (`agents/majordomo/prompt.template.md:31-50`).

## Mixed / Hybrid

Most serious systems are hybrids. AutoGPT combines autonomous loop, tool execution, parallel tool calls, and alternate prompt strategies (`/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:266-460`, `/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:499-557`, `/git/github.com/Significant-Gravitas/AutoGPT/classic/original_autogpt/autogpt/agents/agent.py:588-633`). AutoGen combines group chat runtime with multiple speaker-selection policies and nested team agents (`/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/teams/_group_chat/_base_group_chat_manager.py:86-193`, `/git/github.com/microsoft/autogen/python/packages/autogen-agentchat/src/autogen_agentchat/agents/_society_of_mind_agent.py:168-250`). LangChain docs explicitly say multi-agent patterns can be mixed, including subagents invoking custom workflows or router agents (LangChain, "Multi-agent", accessed 2026-05-04, https://docs.langchain.com/oss/python/langchain/multi-agent/index). Criopolis is already hybrid: council deliberation, tool-agent execution, bead cascade, mail notifications, mayor synthesis, and majordomo first-pass digestion (`agents/mayor/prompt.template.md:73-130`, `agents/majordomo/prompt.template.md:3-108`, `agents/researcher/prompt.template.md:91-117`).
